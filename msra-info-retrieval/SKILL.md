# 多源信息检索与分析

## 概述

本 Skill 面向科大讯飞 AI 开发者大赛 MSRA 赛道，提供多源异构信息的智能检索、融合分析与结构化输出能力。适用于学术研究、技术调研、竞品分析、政策研究等需要从多个信息来源整合知识并给出可溯源结论的场景。

## 触发条件

满足以下任一条件时激活本 Skill：

1. **关键词触发**：用户输入中包含"信息检索"、"多源分析"、"文献综述"、"信息融合"、"深度搜索"、"多文档分析"、"知识图谱"、"实体抽取"、"答案溯源"、"跨源比对"、"综合调研"、"多源异构"、"信息抽取"等关键词。
2. **意图触发**：用户明确要求从多个来源收集、整合、对比或分析信息，例如"帮我调研一下 XX 技术的最新进展"、"对比这几篇文档中关于 XX 的说法"。
3. **场景触发**：用户提供了多份文档/多个网页链接/多个知识片段，并要求进行综合分析、摘要或问答。

## 操作步骤

### 场景一：智能检索与改写

当用户提出一个信息查询需求时：

1. **查询理解与改写**
   - 分析用户原始查询意图，识别核心实体、时间范围、领域约束
   - 将模糊查询改写为精确检索语句（同义词扩展、领域术语标准化）
   - 若查询过于宽泛，主动向用户确认细化方向

2. **多源检索策略**
   - 根据查询主题确定优先检索的信息源类型（学术论文 > 权威网站 > 百科 > 社区论坛）
   - 对每个信息源制定适配的检索策略（关键词组合、布尔逻辑、领域过滤）
   - 记录每个检索结果的来源、时间、URL/出处标识

3. **结果排序与去重**
   - 按相关性、来源权威性、时效性三维排序
   - 去除内容高度重复的结果，保留信息量最大的版本

### 场景二：多文档融合分析

当用户提供了多份文档或多个信息来源时：

1. **单文档预处理**
   - 对每个文档进行关键信息提取（主题、核心论点、数据指标、结论）
   - 标注文档元信息（来源、作者、发布时间、文档类型）

2. **跨文档信息对齐**
   - 识别不同文档中讨论的相同/相关主题
   - 建立跨文档的实体-论点映射表
   - 检测信息一致性与矛盾点

3. **信息融合与矛盾消解**
   - 对一致信息进行合并强化，标注共识强度（多源验证/单源声称）
   - 对矛盾信息依据来源可信度和时效性进行判断，给出消解理由
   - 参考 `reference.md` 中的矛盾消解策略

4. **结构化输出**
   - 按主题维度组织融合后的信息
   - 每条信息必须标注来源出处
   - 使用 `templates/research-summary.md` 模板生成报告

### 场景三：长文档摘要

当用户需要对长文档进行摘要时：

1. **文档结构分析**
   - 识别文档的逻辑结构（标题层级、段落划分、章节关系）
   - 定位核心章节和关键段落

2. **多粒度摘要生成**
   - **一句话摘要**：50 字以内概括文档核心结论
   - **关键发现摘要**：3-5 条核心发现，每条 50-100 字
   - **结构化摘要**：按章节/主题维度的分段摘要

3. **摘要质量校验**
   - 确认摘要未引入原文未提及的信息
   - 确认关键数据点和结论未被遗漏

### 场景四：实体关系抽取

当用户需要从文本中提取结构化知识时：

1. **实体识别**
   - 识别人名、地名、机构名、时间、数值、术语等各类实体
   - 可调用 `scripts/text_analyzer.py` 辅助识别

2. **关系抽取**
   - 识别实体间的语义关系（归属、关联、因果、时序、对比等）
   - 参考 `reference.md` 中的实体关系类型参考表

3. **知识结构化输出**
   - 以表格或列表形式呈现实体-关系三元组
   - 对关键关系提供原文依据

## 输出模板说明

本 Skill 的标准输出报告模板位于 `templates/research-summary.md`，包含以下结构：

- **研究问题**：明确本次分析的核心问题
- **信息来源列表**：每个来源标注类型、可信度等级、时效性
- **核心发现**：3-7 条关键发现，按重要性排序
- **详细分析**：按主题分层的展开分析
- **矛盾点说明**：不同来源间的信息冲突及消解理由
- **溯源引用**：每条结论对应的原始出处
- **结论与建议**：综合判断和后续行动建议

## 已知限制

1. **不直接访问实时数据库**：本 Skill 不建立数据库连接，依赖用户提供的信息源或 Agent 自身的检索能力。
2. **不使用付费 API**：不包含需要 API 密钥的第三方服务调用。
3. **信息时效性依赖来源**：分析结果的时效性取决于输入信息源的发布时间，无法保证实时更新。
4. **语言能力边界**：对中文信息的处理能力优于其他语言，英文信息可处理但可能存在精度损失。
5. **矛盾消解的局限性**：当多个权威来源存在根本性矛盾且缺乏裁决依据时，只能呈现各方观点并标注争议状态，无法给出确定性结论。
6. **文本分析工具的精度**：`scripts/text_analyzer.py` 使用基于规则和统计的轻量方法，在特定领域（如医学、法律）的实体识别精度可能不足，建议作为辅助工具使用。
