# 多源信息分析参考知识

本文档为多源信息检索与分析 Skill 提供参考知识框架，包括信息可信度评估、信息源分类、实体关系类型和矛盾消解策略。

---

## 一、信息可信度评估框架

### 1.1 评估维度

| 维度 | 说明 | 评分标准 |
|------|------|----------|
| **来源权威性** | 发布机构的学术/行业地位 | 顶级期刊 > 行业会议 > 权威机构报告 > 知名媒体 > 一般网站 > 个人博客 |
| **同行评审** | 内容是否经过独立专家审核 | 已评审 > 预印本 > 未评审 |
| **数据可验证性** | 结论是否有原始数据/实验支撑 | 有完整实验数据 > 有部分数据 > 仅定性描述 > 无数据 |
| **时效性** | 信息发布时间与研究主题的匹配度 | 1年内 > 1-3年 > 3-5年 > 5年以上 |
| **利益相关性** | 发布者是否存在潜在利益冲突 | 独立第三方 > 低利益相关 > 高利益相关 |
| **引用广泛度** | 被其他可信来源引用的次数 | 高被引(>100) > 中等被引(10-100) > 低被引(<10) > 未被引用 |

### 1.2 可信度等级定义

| 等级 | 标识 | 典型来源 | 使用建议 |
|------|------|----------|----------|
| **A 级（极高）** | ★★★★★ | 顶级同行评审期刊(Nature/Science/Lancet)、政府官方统计、国际标准组织 | 可作为核心论据 |
| **B 级（高）** | ★★★★☆ | 二级期刊论文、头部咨询机构报告(Gartner/McKinsey)、权威媒体报道 | 可作为重要支撑 |
| **C 级（中）** | ★★★☆☆ | 行业会议论文、企业技术报告、专业书籍、主流媒体深度报道 | 参考使用，需交叉验证 |
| **D 级（低）** | ★★☆☆☆ | 技术博客、科普文章、自媒体、行业论坛 | 仅作为背景参考 |
| **E 级（极低）** | ★☆☆☆☆ | 匿名来源、社交媒体帖子、未经验证的用户生成内容 | 谨慎使用或不予采信 |

## 二、常见信息源分类

### 2.1 按信息类型分类

| 类型 | 代表来源 | 特点 | 典型可信度 |
|------|----------|------|-----------|
| **学术论文** | IEEE, ACM, Springer, CNKI, arXiv | 系统性强、数据详实、经评审 | A-B 级 |
| **专利文献** | USPTO, WIPO, CNIPA | 技术细节准确、法律状态明确 | B 级 |
| **政府/国际组织文件** | WHO, UN, 各国政府网站 | 权威性高、数据权威 | A 级 |
| **行业报告** | Gartner, IDC, 艾瑞, 信通院 | 数据丰富、有分析框架、可能存在商业倾向 | B-C 级 |
| **百科知识** | Wikipedia, 百度百科, 知乎 | 覆盖面广、内容可能滞后、需验证 | C-D 级 |
| **技术文档** | 官方API文档、RFC标准、开发者社区 | 技术准确度高、更新及时 | B 级 |
| **新闻媒体** | 路透社、新华社、专业媒体 | 时效性强、深度可能不足 | C 级 |
| **企业发布** | 官方博客、产品文档、新闻稿 | 偏向自有利、数据选择性呈现 | C-D 级 |
| **社区论坛** | GitHub, Stack Overflow, Reddit | 实用性强、质量参差不齐 | D 级 |

### 2.2 按信息时效性分类

| 分类 | 时间范围 | 适用场景 | 注意事项 |
|------|----------|----------|----------|
| **实时信息** | 24小时内 | 突发事件、市场动态 | 需多源交叉验证 |
| **近期信息** | 1年以内 | 技术趋势、政策变化 | 注意是否已有更新 |
| **中期信息** | 1-5年 | 技术发展脉络、行业分析 | 需确认是否仍然有效 |
| **历史信息** | 5年以上 | 发展历史、长期趋势 | 仅作背景参考 |
| ** timeless** | 无时间限制 | 基础理论、数学公式、定义 | 可直接引用 |

## 三、实体关系类型参考

### 3.1 核心关系类型

| 关系大类 | 关系子类 | 说明 | 示例 |
|----------|----------|------|------|
| **归属关系** | 属于(BELONG_TO) | 实体A属于实体B的组成部分 | "北京" --属于--> "中国" |
| | 包含(CONTAIN) | 实体A包含实体B | "华为" --包含--> "海思半导体" |
| | 子类(IS_A) | 实体A是实体B的子类 | "深度学习" --子类--> "机器学习" |
| **关联关系** | 合作(COOPERATE_WITH) | 两个实体之间存在合作关系 | "OpenAI" --合作--> "Microsoft" |
| | 竞争(COMPETE_WITH) | 两个实体之间存在竞争关系 | "Google" --竞争--> "OpenAI" |
| | 关联(RELATED_TO) | 两个实体存在一般性关联 | "5G" --关联--> "物联网" |
| **因果关系** | 导致(CAUSE) | 实体A导致实体B | "数据泄露" --导致--> "信任危机" |
| | 解决(SOLVE) | 实体A解决/缓解实体B | "RAG" --缓解--> "幻觉问题" |
| | 预防(PREVENT) | 实体A预防实体B | "加密" --预防--> "数据泄露" |
| **时序关系** | 先于(BEFORE) | 事件A发生在事件B之前 | "GPT-3发布" --先于--> "GPT-4发布" |
| | 同时(SIMULTANEOUS) | 两个事件同时发生 | "AlphaGo战胜李世石" --同时--> "AI关注激增" |
| | 后于(AFTER) | 事件A发生在事件B之后 | "ChatGPT发布" --后于--> "InstructGPT发布" |
| **对比关系** | 优于(SUPERIOR_TO) | 实体A在某个维度优于B | "Med-PaLM 2" --优于--> "GPT-4 (医疗)" |
| | 劣于(INFERIOR_TO) | 实体A在某个维度劣于B | "小模型" --劣于--> "大模型 (准确率)" |
| | 等效(EQUIVALENT_TO) | 实体A与B在某个维度等效 | "CNN" --等效--> "ResNet (特定任务)" |
| **属性关系** | 属性(HAS_ATTRIBUTE) | 实体A具有属性B | "GPT-4" --属性--> "多模态" |
| | 数值(HAS_VALUE) | 实体A具有数值B | "Med-PaLM 2" --USMLE分数--> "86.5%" |
| | 状态(HAS_STATUS) | 实体A处于状态B | "项目" --状态--> "进行中" |

### 3.2 关系抽取提示

- 关系抽取时应注意**关系方向**：A 导致 B 与 B 导致 A 是不同的关系
- 关注**关系限定条件**：某些关系仅在特定上下文中成立
- 区分**直接关系**与**间接关系**：优先抽取直接关系
- 对**隐含关系**进行标注：原文未明确表述但可合理推断的关系应标注为"推断"

## 四、矛盾消解策略

### 4.1 矛盾类型分类

| 矛盾类型 | 说明 | 示例 |
|----------|------|------|
| **数据矛盾** | 不同来源给出不同的数值/统计数据 | 来源A说准确率85%，来源B说90% |
| **结论矛盾** | 不同来源对同一问题给出相反结论 | 来源A认为技术成熟，来源B认为不成熟 |
| **时序矛盾** | 对事件发生时间或先后顺序的描述不一致 | 来源A说2023年发布，来源B说2024年发布 |
| **定义矛盾** | 对同一概念的定义或分类标准不同 | 来源A将XX归类为AI，来源B归类为自动化 |
| **归因矛盾** | 对同一结果的原因解释不同 | 来源A归因于算法改进，来源B归因于数据增加 |

### 4.2 消解策略优先级

按以下优先级依次尝试消解矛盾：

**第一优先级：时效性优先**
- 如果两个来源的时间不同，优先采信更新的来源
- 例外：如果较旧的来源是原始一手数据，而较新的来源是二手引用，则优先采信一手来源
- 判断规则：`IF source_A.time > source_B.time AND source_A.is_primary OR NOT source_B.is_primary THEN trust A`

**第二优先级：权威性优先**
- 如果两个来源时效性相近，比较来源的可信度等级
- 同级别来源的矛盾进入下一策略
- 判断规则：`IF credibility(A) > credibility(B) THEN trust A`

**第三优先级：一手来源优先**
- 优先采信直接产生数据的来源（实验报告、原始论文），而非二手引用
- 判断规则：`IF A.is_primary AND NOT B.is_primary THEN trust A`

**第四优先级：样本量/证据量优先**
- 当多个来源给出不同统计数据时，优先采信样本量更大的研究
- 判断规则：`IF sample_size(A) > sample_size(B) THEN trust A`

**第五优先级：并列呈现**
- 当以上策略均无法消解时，将各方观点并列呈现
- 明确标注为"存在争议"或"尚无定论"
- 给出每个观点的支持依据，供用户自行判断

**第六优先级：待验证标注**
- 当矛盾无法通过现有信息消解，且用户需要确定性结论时
- 标注为"需进一步验证"，建议用户查阅更多来源

### 4.3 矛盾消解输出格式

对每个检测到的矛盾，按以下格式输出：

```
矛盾编号：C{n}
矛盾类型：{数据矛盾/结论矛盾/时序矛盾/定义矛盾/归因矛盾}
来源A：{来源名} - {观点摘要}
来源B：{来源名} - {观点摘要}
消解策略：{采用的策略名称}
消解结论：{最终采纳的观点}
消解理由：{详细说明为什么采纳此观点}
残留风险：{低/中/高} - {说明}
```

### 4.4 特殊处理规则

- **系统性偏差检测**：如果所有来源均来自同一利益相关方，标注"存在系统性偏差风险"
- **信息茧房警告**：如果所有来源观点高度一致且缺乏反面证据，标注"可能存在信息茧房效应，建议主动寻找反对意见"
- **版本差异**：对同一来源的不同版本（如论文 preprint vs. 正式版），标注版本差异并采信正式版
